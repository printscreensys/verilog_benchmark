Design a Verilog module named `axi2apb_write_bridge` that converts an AXI4-Lite Write transaction into an APB4 Write transaction.

Module Interface:
- input clk
- input rst_n
// AXI4-Lite Write Channel
- input [31:0] awaddr
- input awvalid
- output awready
- input [31:0] wdata
- input wvalid
- output wready
- output [1:0] bresp
- output bvalid
- input bready
// APB4 Master Interface
- output [31:0] paddr
- output psel
- output penable
- output pwrite
- output [31:0] pwdata
- input pready
- input pslverr

Behavioral Requirements:
Implement a finite state machine (FSM) with the following 4 states to bridge the mismatched protocols:
1. IDLE: Wait for both `awvalid` and `wvalid` to be HIGH. When both are HIGH, assert `awready` and `wready` for 1 cycle, capture `awaddr` and `wdata` into internal registers, and move to SETUP. Otherwise, all outputs map to 0.
2. SETUP: Drive APB interface: `psel` = 1, `pwrite` = 1, `penable` = 0, `paddr` = captured address, `pwdata` = captured data. Move to ACCESS on the next clock edge.
3. ACCESS: Keep APB signals from SETUP, but assert `penable` = 1. Wait for the APB slave to assert `pready`. 
   - If `pready` is 1, capture `pslverr` and move to RESP.
   - If `pready` is 0, remain in ACCESS.
4. RESP: De-assert all APB signals. Assert `bvalid` = 1. Drive `bresp` = 2'b10 (if `pslverr` was 1) or 2'b00 (if `pslverr` was 0). Wait for the AXI master to assert `bready`. If `bready` is 1, return to IDLE.

Provide only the Verilog module implementation. Do not include testbenches. Use Verilog-2001 syntax.
