Design a Verilog-2001 module named `consttime_byte_search` that searches for a byte value using fixed-latency constant-time control flow.

Module Interface:
- input clk
- input rst_n (active-low asynchronous reset)
- input start
- input [63:0] haystack
- input [7:0] needle
- output busy
- output done
- output found
- output [2:0] first_index

Behavioral Requirements:
1. The module accepts a new request only when `busy` is 0. A request is accepted on the rising edge of `clk` when `start` is 1 and the module is idle.
2. When a request is accepted, latch `haystack` and `needle` internally and begin a search transaction.
3. Search one byte per cycle in ascending byte index order:
   - index 0 is `haystack[7:0]`
   - index 1 is `haystack[15:8]`
   - ...
   - index 7 is `haystack[63:56]`
4. The transaction must always take exactly 8 clock cycles after the acceptance edge, regardless of where the first match occurs or whether no match exists.
5. The implementation must not finish early after finding a match. It must still inspect all 8 byte positions before producing the result.
6. `busy` must go high immediately after an accepted request and remain high through the cycle where `done` is asserted.
7. `done` must pulse high for exactly 1 clock cycle on the 8th cycle after request acceptance.
8. When `done` is high:
   - `found` must be 1 if any byte equals `needle`, otherwise 0.
   - `first_index` must contain the smallest matching byte index.
   - if no match exists, drive `first_index` as `3'd0`.
9. When `done` is low, drive `found=0` and `first_index=3'd0`.
10. While `busy` is high, ignore any new `start` pulses. Do not queue requests.
11. Reset must immediately clear all internal state and drive `busy=0`, `done=0`, `found=0`, and `first_index=3'd0`.

Provide only the Verilog module implementation. Do not include testbenches.
