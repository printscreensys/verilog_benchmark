Coverage-driven verification task: close functional coverage for the protected
`queue_credit_tracker` DUT.

Goal:
- Reach at least 90% functional coverage within 6 iterations.
- Each iteration replays the same tests twice. Flaky or nondeterministic tests
  are penalized.

Suggested workflow:
1. Prepare a workspace:
   `python3 domains/12_cdv/runner.py prepare task_121 <workspace>`
2. Edit `<workspace>/tests/test_sequences.py`
3. Run one iteration:
   `python3 domains/12_cdv/runner.py run <workspace>`
4. Inspect `<workspace>/.cdv/reports/iteration_XX/report.json`
5. Repeat until the coverage target is met or the budget is exhausted.

DUT summary:
- `cfg_limit` sets the queue depth limit as `cfg_limit + 1`, so the legal
  capacities are 1..4 entries.
- `push` without `pop` increments occupancy unless the queue is already full,
  in which case `overflow_pulse` asserts for one cycle and `count` does not
  change.
- `pop` without `push` decrements occupancy unless the queue is empty, in which
  case `underflow_pulse` asserts for one cycle and `count` does not change.
- `flush` clears occupancy to zero and has priority over all other operations.
- When `push` and `pop` are both asserted in the same cycle:
  - if the queue is empty, push wins and occupancy increments by one
  - if the queue is full, pop wins and occupancy decrements by one
  - otherwise occupancy stays unchanged
