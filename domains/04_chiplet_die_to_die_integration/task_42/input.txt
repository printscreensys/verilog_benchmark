Design a standalone Verilog-2001 module named `alloc_two`.

This module is a two-output channel allocator for a NoC input port. Include all needed macro definitions and a small FIFO module named `fifo_NoD_wrapper` so the answer compiles standalone.

Parameters:
- `CHANNEL_ID`: one of `LEFT`, `RIGHT`, `UP`, `DOWN`.
- `ROUTER_ID_X`, `ROUTER_ID_Y`: current router coordinates.

Interface:
- Input stream: `data_i[DATA_WIDTH-1:0]`, `valid_i`, `ready_o`.
- Output A stream: `A_valid_o`, `A_ready_i`, `A_data_o[DATA_WIDTH-1:0]`.
- Output B stream: `B_valid_o`, `B_ready_i`, `B_data_o[DATA_WIDTH-1:0]`.
- Clock/reset: `clk`, active-low async reset `rstn`.

Required behavior:
- Push input flits into an internal FIFO when `valid_i && ready_o`.
- `ready_o` is high when the FIFO is not full.
- Extract route ID bits from each flit and split them into destination X and Y coordinates.
- For `LEFT`, A means destination Y is greater than `ROUTER_ID_Y`; B means equal Y.
- For `RIGHT`, A means destination Y is less than `ROUTER_ID_Y`; B means equal Y.
- For `UP`, A means destination X is greater than `ROUTER_ID_X`; B means equal X.
- For `DOWN`, A means destination X is less than `ROUTER_ID_X`; B means equal X.
- At packet head, choose A/B from the current flit route. Once any flit fires, lock that choice until a tail flit fires.
- A flit fires on an output when that output valid and ready are both high.
- Drive `fifo_ready` from the selected output ready signal.
- Both output data buses mirror the current FIFO output flit.
